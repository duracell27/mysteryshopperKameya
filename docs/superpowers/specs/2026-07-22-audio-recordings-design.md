# Аудіозаписи перевірки — Специфікація

**Дата:** 2026-07-22  
**Статус:** Затверджено

---

## Мета

Додати можливість прив'язувати MP3-аудіозаписи перевірки до анкет таємного покупця. Адмін додає записи (через файл або URL), вони зберігаються локально на сервері. Як адмін, так і працівник можуть відтворювати записи у своїх відповідних переглядах.

---

## Вимоги

- До однієї анкети можна прив'язати **декілька** аудіозаписів
- Два способи додавання: **завантаження файлу** (провідник) або **пряме URL** (сервер викачує файл локально)
- Файли зберігаються **локально** на сервері (`backend/uploads/audio/`)
- Формат файлів: **MP3**
- **Без обмеження розміру** файлу
- Мітка генерується **автоматично**: `"{ПІБ працівника} - {Місяць} {Рік}"`
- Адмін може **видаляти** окремі записи
- Додавання можливе як **під час завантаження нової анкети**, так і **до вже існуючої**
- **Працівник** бачить і відтворює записи у своєму перегляді анкети

---

## Архітектура

### Зберігання файлів

- Директорія: `backend/uploads/audio/`
- Назва файлу на диску: `{uuid4}.mp3` (уникає колізій, не розкриває оригінальні назви)
- Директорія створюється автоматично при старті сервера (`fs.mkdirSync(..., { recursive: true })`)

### Доступ до файлів

Файли **не** роздаються через `express.static`. Використовується захищений маршрут зі стримінгом і перевіркою JWT. Токен передається через query-параметр `?token=` (необхідно для HTML5 `<audio src>`, який не підтримує кастомні заголовки).

## Модель даних

### Новий субдокумент `IAudioRecording` (backend/src/models/Report.ts)

```ts
interface IAudioRecording {
  label: string;        // "Іваненко О. - Листопад 2025"
  filename: string;     // "3f2a1b4c-....mp3" — ім'я на диску
  originalName: string; // оригінальна назва файлу
  uploadedAt: Date;
}
```

Додається до `IReport`:
```ts
audioRecordings?: IAudioRecording[];
```

### Зміни у frontend/src/types.ts

```ts
export interface AudioRecording {
  _id: string;
  label: string;
  filename: string;
  originalName: string;
  uploadedAt: string;
}
```

Додається до `AuditResult`:
```ts
audioRecordings?: AudioRecording[];
```

## API маршрути

Всі маршрути захищені `authMiddleware`. Лише адмін може додавати/видаляти. Адмін або власник звіту можуть стримити.

### `POST /api/reports/:id/audio/upload`
- Прийом MP3 через `multipart/form-data` (multer з `diskStorage`)
- Валідація: mime `audio/mpeg` або розширення `.mp3`
- Генерує uuid-ім'я, зберігає в `uploads/audio/`
- Додає субдокумент до `report.audioRecordings`
- Відповідь: оновлений звіт

### `POST /api/reports/:id/audio/url`
- Приймає `{ url: string }` у тілі
- Сервер завантажує файл через `https.get` (таймаут 30 сек)
- Перевіряє `Content-Type` відповіді на `audio/mpeg` або `audio/mp3`
- Зберігає з uuid-ім'ям у `uploads/audio/`
- Відповідь: оновлений звіт

### `GET /api/reports/:id/audio/:audioId/stream?token=`
- Перевірка JWT з query `token`
- Авторизація: адмін або власник звіту (`report.userId`)
- Читає файл через `fs.createReadStream`
- Підтримка `Range` → `206 Partial Content` (потрібно для перемотування)
- Без `Range` → `200` з повним файлом
- Якщо файл відсутній на диску → `404`

### `DELETE /api/reports/:id/audio/:audioId`
- Тільки адмін
- `fs.unlink` файлу (помилка ігнорується якщо файл відсутній)
- `$pull` субдокументу з `audioRecordings`
- Відповідь: оновлений звіт

## Фронтенд

### Генерація мітки (утиліта)

```ts
function buildAudioLabel(employeeName: string, month: number, year: number): string {
  return `${employeeName} - ${MONTHS_UK[month - 1]} ${year}`;
}
```

### ReportsUploadView (нова анкета)

На кроці `preview` — нова секція **"Аудіозаписи"**:
- Список вже доданих `pendingAudios: File[]` з можливістю видалити до підтвердження
- Кнопка **"Обрати файл"** (`<input type="file" accept=".mp3">`)
- Поле URL + кнопка **"Додати"**
- Після `POST /confirm` (отримали `report._id`) — послідовні `POST /:id/audio/upload` для кожного файлу
- Якщо хоч одне завантаження не вдалося — попередження після збереження (анкета вже збережена, не скасовується)

### AdminReportsListView (існуюча анкета)

У деталях анкети — секція **"Аудіозаписи"**:
- Перелік існуючих записів: мітка + кнопка видалити (з підтвердженням)
- Ті ж два варіанти додавання (файл / URL)
- Після будь-якої дії — оновлює локальний стан `selected`

### MyReportsView (працівник)

У перегляді анкети — секція **"Аудіозаписи"** (відображається тільки якщо `audioRecordings.length > 0`):
- Кожен запис: мітка + `<audio controls src="/api/reports/:id/audio/:audioId/stream?token={jwt}">`
- Токен береться з `localStorage` / `AuthContext`

## Обробка помилок

| Сценарій | Поведінка |
|---|---|
| URL недоступний або не MP3 | 400 з описом помилки |
| Таймаут завантаження за URL (>30 сек) | 408 |
| Файл відсутній на диску при стримінгу | 404 (запис у БД не чіпаємо) |
| Некоректний `Range` заголовок | Повертаємо повний файл (200) |
| `fs.unlink` при видаленні не вдалося | Ігноруємо, субдокумент все одно видаляється |
| Завантаження аудіо після `confirm` не вдалося | Попередження у UI, анкета збережена |
| Стримінг не-власником і не адміном | 403 |

---

## Безпека

- JWT в query-параметрі `?token=` тільки для стримінг-маршруту
- Усі інші маршрути використовують стандартний `Authorization: Bearer` header
- Файли ізольовані від публічного доступу (не через `static`)
- Назви файлів на диску — uuid, не пов'язані з іменами користувачів
